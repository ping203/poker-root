This package contains the API for the multi table tournament (MTT) support in Firebase.

Package Specification

A tournament implementation is responsible for maintaining a set of tables and for the seating of tournament players on the tables.

The deployed tournament have a common logic part and 1..* instances. The logic is stateless and the instances are stateful.

The state of a tournament basically includes a set of tables and optional custom data.

Tournament instances are created using an activator and a creation participant. Where the activator is responsible for "when" the participant is responsible for "how".
